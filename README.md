# 梦境 SVN 管理器

这是一个给游戏策划使用的 SVN 简化工具，目标是把常用操作做成少量中文按钮，降低误操作。

## 第一版功能

- 选择本地 SVN 工作目录
- 保存多个本地 SVN 工作副本，并通过“本地库”下拉框切换
- 保存/移除本地库后，左侧“本地库”树会同步刷新，新增 `version` 等分支工作副本会立即显示
- 左侧“本地库”树会高亮当前库，并在悬停时显示完整本地路径
- 从 SVN 地址检出工作副本
- 拉取最新，对应 `svn update`
- 拉取最新前会先检查本地改动；如果还有未提交内容，会提示“建议先查看/提交/备份再更新”，避免策划不清楚风险就更新出冲突
- 状态栏会定时检查远端最新提交；如果服务器版本高于当前工作副本，会显示“远端有新提交 rxxxxx”，点击状态栏文字可立即重新检查
- 查看本地改动，对应 `svn status`
- 更新后如果出现冲突，会提示 `.mine`、`.r旧版本`、`.r新版本` 是 SVN 冲突辅助文件
- 勾选文件并提交，对应 `svn commit`
- 提交前会使用最新 `svn status` 做安全检查，并弹出提交预览，确认本次提交文件清单后才执行提交
- 如果工作副本仍有冲突，会自动跳到“冲突”页并阻止提交
- 提交会拦截 `.mine`、`.r版本号`、`.tmp`、`.bak`、`.orig` 等辅助/临时/备份文件
- 提交成功后会刷新状态和 History，并自动定位到当前工作副本的新版本位置
- 对未加入版本控制的文件，提交前自动执行 `svn add`
- 冲突文件会标红并禁止提交
- “全部文件”页签用树形目录显示工作副本内的所有文件，包括 `.xml`、`.lua` 等，即使没有本地改动也能查看
- “全部文件”树支持搜索文件名/路径、仅显示改动文件、展开、折叠和刷新
- 文件树使用文件夹/文件类型图标，并强制文件夹排在文件前面，浏览目录时更接近资源管理器
- 树视图改为列表式导航风格：更高行距、隐藏默认虚线、整行选中、去掉边框，并降低状态颜色饱和度
- 文件树会显示 SVN 状态标记：`M` 修改、`A` 新增、`D` 删除、`?` 未加入、`!` 缺失、`C` 冲突
- 文件树会保存每个本地库的展开/折叠状态，下次打开会恢复
- 文件树右键菜单支持：打开文件、打开所在目录、查看差异、查看冲突、文件/文件夹历史、加入版本控制、标记冲突已解决
- “全部文件”树支持资源管理器式多选：普通点击单选，`Ctrl` 切换多选，`Shift` 连续选择；右键会优先作用于当前高亮的多个文件/文件夹
- 文件/文件夹历史窗口右侧会以资源管理器式 Changed files 树展示该提交改动项，选择具体文件后可直接预览该文件在当前提交里的差异
- 查看差异会自动判断文件类型：Excel 2003 XML / `.xls` 使用单元格差异，`.lua` 等文本文件使用按 hunk 分组的红绿文本差异
- Diff preview 顶部会显示当前文件、差异范围和两端版本，例如 `SVN BASE -> 本地工作副本`、`r57170 -> r57171`
- History 和文件/路径历史里的 Diff preview 会异步加载；连续切换提交或文件时会取消上一轮预览，并缓存最近看过的差异，减少大 XML / Lua 文件反复计算造成的卡顿
- 文本差异支持搜索行号/内容，并可切换“全部 / 只看改动 / 只看新增 / 只看删除”，hunk 标题会明确显示变更块大致行号
- Excel 差异支持搜索 ID / 字段 / 新旧值 / 单元格，并可单独按 ID 或字段过滤，主预览、独立差异窗口和冲突查看窗口使用同一套体验
- 工具栏和右键菜单支持“外部对比/合并”，会优先调用工程内置的 `SVNManager\ExternalTools\分久必合\分久必合.exe`
- 普通 XML / XLSX / CSV 文件可用分久必合对比 `SVN BASE -> 本地文件`
- 查看冲突会识别 `.mine`、`.r旧版本`、`.r新版本`，显示“我的版本 vs 服务器版本”“当前工作文件 vs 服务器版本”等对比页，不做自动合并
- `.mine`、`.r版本号` 等 SVN 冲突辅助文件不会再作为普通改动显示在 File Status / Uncommitted / 全部文件树中，避免误提交或误点
- 对冲突 XML 表格执行“查看差异”时，默认比较“我的版本 vs 服务器版本”，不再拿带冲突标记的当前文件做 Excel 解析
- 冲突 XML 表格可一键用分久必合打开“我的版本 vs 服务器版本”，SVNManager 只负责准备文件，不自动改动或合并
- 工具栏和文件树右键提供“冲突处理”流程：打开分久必合、定位当前冲突文件目录、等待人工确认保存结果后执行 `svn resolve` 并刷新状态
- 独立“冲突”页签只列出当前冲突文件，每行提供“打开合并 / 打开目录 / 标记解决”三个动作，避免在全文件树里查找冲突
- File Status、冲突、History 页签会显示数量角标，例如 `File Status(5)`、`冲突(2)`、`History(80)`
- File Status、冲突、History 面板顶部都有独立刷新按钮，不需要回到主工具栏刷新
- 顶部工具栏只保留常用主流程：本地库、保存/移除、拉取最新、提交、打开目录、更多操作；差异、外部对比、冲突处理、文件历史等低频动作集中到“更多操作”和右键菜单
- 关键操作会写入本地操作日志，包括拉取最新、提交、标记冲突已解决、打开外部合并工具；可在“更多操作 -> 打开操作日志”查看
- 左侧栏只保留本地库列表，去掉 WORKSPACE 节点
- 查看 Excel 2003 XML `.xml` 和传统 `.xls` 单元格差异
- Excel 差异可以从“全部文件”树里直接选择 `.xml` 表格文件，也可以从改动列表里选择
- 查看选中文件最近提交历史，对应 `svn log -v`
- “History”页签默认显示整个仓库最近提交历史，不需要先选中文件；历史解析使用普通文本日志，避免 SVN XML 日志异常导致崩溃
- History 是默认主视图，列结构接近 SourceTree：Graph、Description、Date、Author、Commit
- History 支持按提交说明、作者、版本号、改动文件路径搜索
- History 搜索支持结构化条件：`file:skill_data.xml`、`author:张三`、`rev:57100-57120`、`id:260428`，双击搜索结果会优先定位到命中的改动文件
- History 会自动选中当前工作副本所在版本；双击提交会定位到本次提交的第一个改动文件
- 当前工作副本位置使用 `svnversion` 的最高版本判断，能正确处理提交后的混合版本工作副本，例如根目录仍是旧版本但刚提交的文件已经是新版本
- History 提交列表支持右键菜单：定位改动文件、复制版本号、刷新历史、回退工作副本到选中版本；回退使用 `svn update -r`，只影响本地工作副本，不会自动提交或改写服务器历史
- History 支持多选一段提交；下方会汇总这段提交的 Changed files，选中文件后显示从范围开始前到范围结束后的最终差异
- 单个文件的“文件历史”会打开独立窗口，不再替换主 History 仓库历史；窗口内显示该文件提交列表、提交详情和该版本文件差异
- History 顶部显示 `Uncommitted changes`，日志中会用 `[当前工作副本]` 标记当前本地版本所在提交
- 选中 History 中的提交后，下方详情会显示该次提交的 Changed files 列表，来源于 `svn log -v`
- History 下方改为 SourceTree 风格左右分栏：左侧选择本次提交改动文件，右侧显示该文件改动内容
- 主窗口默认给 History 的 Changed files / Diff preview 区域更多高度，提交说明和命令输出保持较小固定高度
- 提交说明不再常驻主界面，只在点击“提交选中文件”后进入提交确认窗口时显示和编辑
- 提交确认窗口支持搜索文件，并可在提交前取消某些文件；最终只提交窗口里仍勾选的文件
- 提交确认窗口会显示不可提交文件的阻止原因，例如冲突、缺失、SVN 冲突辅助文件或临时/备份文件，并禁止勾选这些文件
- 提交文件差异预览支持 Excel 表格单元格差异和 `.lua` 等文本行差异
- Excel 差异会显示字段名（默认取第 2 行列名）和 ID（默认取第 1 列），方便定位是哪条配置被改动
- History 的 Changed files 树支持双击打开文件；选中提交时会打开该提交的历史版本，多选一段提交时会打开范围结束版本，未提交改动则打开本地文件
- History 的 Changed files 右键支持打开文件、打开所在目录、用分久必合对比、查看文件历史、复制仓库路径；外部对比同样支持单个提交和多选提交范围
- 会保存窗口大小/位置、最大化状态、左侧本地库宽度、History 上下分割、Changed files 与 Diff preview 左右分割，以及最后打开的页签
- 文件状态解析也使用普通文本 `svn status`，避免 SVN XML 状态输出异常导致崩溃
- SourceTree 风格布局：左侧工作区，中间文件状态和历史面板，下方提交说明与命令输出

## 默认 SVN 地址

```text
svn://192.168.6.20:13690/xdqd_xml/trunk
```

如果实际仓库使用 `http://` 或 `https://`，在界面里直接改掉即可。

## 运行

开发运行：

```powershell
dotnet run --project .\SVNManager\SVNManager.csproj
```

编译：

```powershell
dotnet build .\SVNManager\SVNManager.csproj
```

编译后的程序在：

```text
SVNManager\bin\Debug\net8.0-windows\SVNManager.exe
```

## 当前边界

- 第一版先支持 SVN，Git 可以后续按同样结构接入。
- 暂不自动处理冲突，冲突需要人工解决。
- 本地缺失文件暂不自动提交删除，避免策划误删配置。
- Excel 差异当前按单元格显示字段名、ID、“旧值 / 新值”，不比较样式、颜色、批注、筛选条件。
